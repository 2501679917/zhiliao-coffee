# 通用实体门店网站技能设计

日期：2026-07-13  
技能名：`build-local-business-website`

## 目标

创建一个安装于个人 Codex 技能目录的混合型技能，用于改造或新建咖啡馆、餐厅、茶室、民宿、工作室、零售店等实体门店网站。技能复用可靠流程与确定性工具，同时根据每家门店的真实素材、品牌气质和业务目标重新设计，不套用固定视觉模板。

## 触发范围

当用户要求创建、改造或完善实体门店网站，并提供旧网站、品牌资料、实拍照片、菜单、服务项目或门店信息时触发。

技能不默认加入预约、菜单或留言功能。它必须先理解门店类型、展示目标和用户描述，再选择需要的模块，例如：

- 品牌与空间展示
- 菜单、商品或服务项目
- 地址、营业时间与地图
- 预约、询价、留言或联系表单
- 管理后台与内容更新能力
- 临时公网演示链接
- 正式部署建议

## 核心工作流

1. 勘察现有项目、技术栈、版本控制和素材目录。
2. 核实地址、营业时间、价格、服务项目等事实，不传播旧站中的虚构内容。
3. 逐张审查素材，区分可直接使用、仅供参考、含人物、含水印/截图界面和禁止使用素材。
4. 通过 brainstorming 流程提出 2–3 个视觉方向并获得用户确认。
5. 为所选方向建立色彩、字体、图片、版式、交互和移动端规则。
6. 根据需求选择功能模块，不添加未请求的业务系统。
7. 形成规格与实施计划后再修改项目。
8. 实施本地图片、响应式布局、触摸/键盘可访问交互和真实内容。
9. 对表单或接口实现成功、验证、超时、后端不可用等诚实状态。
10. 自动检查外部图库、人物文件、水印风险、错误营业时间、资源缺失和敏感接口。
11. 验证桌面、手机、减少动态设置和失败回退。
12. 提交 Git，并按用户需要创建临时 HTTPS 演示链接。

## 真实性与图片规则

- 优先使用门店真实照片，不用无关图库照片冒充门店。
- 用户未授权人物公开使用时，排除人物及可识别人物局部。
- 截图界面、水印、账号信息、日期和地图标识不得进入成品图。
- AI 图片编辑允许改善构图、视角、光线、清晰度和材质，但必须保留真实门店身份。
- 不得虚构不存在的房间、座位、建筑结构、景观、产品或服务。
- 生成图必须与原图并排复核；生成服务需要登录、订阅或费用时明确报告，不能静默降级。
- 品牌标识优先从真实招牌或用户提供的源文件提取，不用普通字体伪造独特字形。

## 资源职责

### `SKILL.md`

保持精简，定义触发后的顺序、设计确认门槛、需要读取的参考文件、脚本调用时机以及完成标准。

### `inventory_media.py`

扫描用户指定素材目录，输出文件名、扩展名、尺寸、大小、修改时间和 SHA-256 清单。脚本只读源目录，结果写入项目内用户指定位置。

### `audit_site.py`

扫描网站文本与资源引用，报告：

- 外部图片 URL
- 失效的本地图片路径
- 已知禁止文件名
- 旧营业时间或用户指定禁用文本
- 截图/水印素材被生产页面引用
- 公开的敏感数据列表接口

脚本只报告，不自动删除或改写用户文件。

### `start_demo_tunnel.ps1`

验证本地服务后启动 Cloudflare Quick Tunnel，输出临时 HTTPS URL、进程 ID、日志位置和注意事项。脚本必须提醒链接无长期在线保证、电脑需保持运行，并在分享前要求检查敏感接口。

### 参考文档

- `discovery-and-content.md`：事实核验、需求问题和门店内容结构。
- `visual-and-media.md`：视觉方向、真实性、人物、水印、AI 编辑和交互规则。
- `feature-selection.md`：按业务描述选择展示、菜单、商品、预约、留言或后台。
- `delivery-and-security.md`：测试、Git、临时演示、数据隐私和正式部署边界。

## 错误与阻塞处理

- 素材不可读：报告具体路径，继续处理其他素材。
- 图片工具或模型不可用：保留真实原图并报告限制，不生成虚假替代品。
- 外部生成服务未登录或要求订阅：停止该增强步骤，继续不依赖它的工作。
- Git 仓库不存在：在用户确认后初始化；远程仓库不明确时不得推送到猜测的仓库。
- 临时隧道失败：检查本地服务、进程、端口和公网返回状态；只交付经过 HTTP 200 验证的链接。
- 敏感查询接口公开：在创建演示链接前关闭或保护接口。

## 验证标准

- `quick_validate.py` 通过技能格式检查。
- 三个脚本均提供 `--help` 或清晰参数说明。
- 媒体清单脚本在临时样例目录上输出稳定结果且不修改源文件。
- 网站审计脚本能检测外部图片、缺失资源和禁用文本，并以非零状态表示发现问题。
- 隧道脚本在无 Cloudflared 时给出安装/路径提示，不静默失败。
- 技能正文无 `TODO`、`TBD` 或项目专属路径。
- 技能可在不同门店类型上触发，且不会固定生成咖啡馆文案或预约功能。

## 非目标

- 不内置固定网站模板或统一品牌风格。
- 不自动创建收费云资源、域名或生产部署。
- 不默认开发管理后台、电商、支付、会员或在线点单。
- 不存储任何客户门店照片、账号凭据、个人信息或 API 密钥。
